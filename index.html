Procedimiento en MóduloAtención al Cliente
1.- Primero, una de las situaciones que nos podemos encontrar es que haya bloqueos en la aplicación,
si se puede acceder a la máquina vía ssh o web pero vemos lentitud en cargar algunas partes o que
tarda en resolver, podemos realizar lo siguiente:
Reinicio de tomcat:
1. Acceder al servidor correspondiente vía ssh (consultar datos servidor en Wiki)
2. Consultamos los procesos para verificar que están iniciados: ps aux | grep tomcat
3. Ir al directorio de la aplicación para apagarla: cd /opt/tomcat-6.0.33/bin
4. Paramos el proceso: sudo sh shutdown-profile.sh
5. Consultamos los procesos para verificar que se ha finalizado el apache: ps aux | grep tomcat
6. Si resulta que no hay procesos, ejecutamos el comando: sudo sh startup-profile.sh
7. Consultamos los logs para ver que está todo OK:
8. tail -fn 2000 /opt/tomcat-6.0.33/profile/logs/catalina.out
Reinicio de jboss:
1. Acceder al servidor correspondiente vía ssh (consultar datos servidor en Wiki)
2. Consultamos los procesos para verificar que están iniciados: ps aux | grep jboss
3. Paramos el proceso: sudo /etc/init.d/jboss_clientprofile.sh stop
4. Consultamos los procesos para verificar que se ha finalizado el apache: ps aux | grep jboss
5. Si resulta que no hay procesos, ejecutamos el comando:
sudo /etc/init.d/jboss_clientprofile.sh start
6. Consultamos los logs para ver que está todo OK:
tail -fn 1000 /opt/jboss-4.2.3.GA/server/default/log/server.log

Reinicio de la aplicación
1. Acceder al servidor mediante ssh con nuestro usuario.
2. En el caso de encontrarnos en un servidor de filaunica debemos ejecutar el comando en la
consola sudo su filaunica, si por el contrario nos encontramos en un servidor de aforo
debemos ejecutar el comando sudo su aforoapli.
3. Podemos verificar el estado de la aplicación ejecutando el comando sudo systemctl status
aforo.service fijandonos en la linea Active para verificar el estado del servidor en el caso de
ticketboxer o aforo siendo el comando sudo systemctl status filaunica.service en el caso de
la aplicación de fila única.
4. Si necesitamos parar la aplicación ejecutamos el comando sudo systemctl stop aforo.service
para ticketboxer y aforo para el caso de fila única el comando sería sudo systemctl stop
filaunica.service.
5. Para iniciar la aplicación de ticketboxer y aforo ejecutamos el comando sudo systemctl start
aforo.service para efectuar esta operación en filaunica ejecutamos el comando sudo systemctl
start filaunica.service.
